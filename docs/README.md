# Meme Stealer — Документация

## Обзор

Meme Stealer — это пользовательский бот Telegram (юзербот), который отслеживает несколько каналов на предмет нового медиаконтента (мемов), удаляет дубликаты с помощью хеширования миниатюр и пересылает уникальный контент в прокси-канал для проверки. Дубликаты или нежелательный контент отправляются в отфильтрованный канал.

### Основные возможности

- **Мониторинг нескольких каналов**: Опрос множества каналов Telegram на наличие новых сообщений
- **Дедупликация**: Использует MD5-хеширование обрезанных миниатюр (~32x32 JPEG) для обнаружения дубликатов
- **Поддержка альбомов**: Обрабатывает сгруппированные сообщения (альбомы Telegram) как единое целое
- **Фильтрация рекламы**: Автоматически отфильтровывает сообщения, содержащие URL-адреса
- **Бот администратора**: Опциональный бот на базе Telegraf для управления каналами и администраторами через команды
- **SOCKS5 прокси**: Весь трафик маршрутизируется через настраиваемый прокси

## Как работает дедупликация

1. **Извлечение миниатюры**: Для каждого сообщения с медиа извлекается обрезанная миниатюра (тип `i`, ~32x32 JPEG), встроенная в ответ Telegram API — загрузка не требуется.
2. **Вычисление MD5**: Байты миниатюры хешируются с помощью MD5.
3. **Проверка базы данных**: Хеш ищется в таблице `minithumbnails`.
4. **Принятие решения**:
   - **Новый хеш** → Пересылка в PROXY-канал, создание записи в базе данных
   - **Существующий хеш** → Пересылка в FILTERED-канал, добавление канала-источника в отслеживание
5. **Обработка альбомов**: Для сгруппированных сообщений миниатюра первого сообщения используется для принятия решения за всю группу, но отслеживаются ВСЕ миниатюры в группе.

### Ограничения

Дедупликация по точному совпадению MD5 хорошо работает для:
- Репостов и пересылок одного и того же сообщения
- Повторных загрузок через Telegram (одинаковое сжатие)

Она НЕ обнаруживает:
- Независимые загрузки одного и того же файла изображения (разное JPEG-сжатие)
- Скриншоты или отредактированные версии
- Изображения с добавленным текстом, водяными знаками или обрезкой
- Немного измененные по размеру версии

См. `docs/perception-hash-plan.md` для ознакомления с планом перехода на перцептивное хеширование (pHash/dHash), которое решит эти ограничения.

## Установка

### Требования

- Node.js 18+
- PostgreSQL 16
- Учетные данные Telegram API (получить на [my.telegram.org](https://my.telegram.org))
- Запущенный SOCKS5 прокси на `127.0.0.1:1081` (или обновите конфигурацию)

### Установка

```bash
# Установка зависимостей
npm install

# Настройка переменных окружения (скопируйте и отредактируйте)
cp .env.example .env

# Запуск миграций базы данных
npx prisma migrate dev

# Заполнение начальными данными (каналы и администраторы)
npm run seed

# Генерация строки сессии (интерактивно)
npx tsx connect.ts
# Вставьте полученную STRING_SESSION в ваш .env файл
```

### Запуск

```bash
# Разработка (с горячей перезагрузкой)
npm run dev

# Продакшн
npm run build
npm start

# Или через PM2
pm2 start ecosystem.config.js
```

## Конфигурация

### Переменные окружения

| Переменная | Описание | Пример |
|----------|-------------|---------|
| `API_ID` | ID Telegram API | `12345678` |
| `API_HASH` | Хеш Telegram API | `abc123...` |
| `STRING_SESSION` | Сериализованная сессия Telegram | `1BVtsOK...` |
| `BOT_TOKEN` | (Опционально) Токен бота Telegraf | `123456:ABC...` |
| `DB_USER` | Имя пользователя PostgreSQL | `postgres` |
| `DB_PASSWORD` | Пароль PostgreSQL | `secret` |
| `DB_NAME` | Имя базы данных PostgreSQL | `postgres` |
| `DB_PORT` | Порт PostgreSQL | `5432` |

### Настройка прокси

Хост и порт SOCKS5 прокси жестко заданы в `src/utils/config.ts`:

```typescript
const SOCKS5_PROXY = { host: '127.0.0.1', port: 1081 };
```

Измените это, если ваш прокси работает по другому адресу.

## Схема базы данных

### Таблицы

| Таблица | Назначение |
|-------|---------|
| `admins` | Авторизованные пользователи бота с ролями (`admin` / `full_access`) |
| `chats` | Отслеживаемые каналы Telegram с состоянием опроса |
| `minithumbnails` | Записи дедупликации (MD5-хеш миниатюр) |
| `minithumbnail_sources` | Связующая таблица: какие каналы публиковали какой контент |
| `stickers` | (Зарезервировано) Отслеживание наборов стикеров для будущей функции |

### Ключевые связи

```
Chat 1 ────< MinithumbnailSource >──── 1 Minithumbnail
```

Каждая `Minithumbnail` может иметь несколько каналов-источников, которые отслеживаются через связующую таблицу `MinithumbnailSource`.

## Команды бота

Команды отправляются опциональному help-боту на базе Telegraf (требуется `BOT_TOKEN`).

### Базовые команды

| Команда | Описание |
|---------|-------------|
| `/start` | Показать приветственное сообщение |
| `/help` | Список доступных команд |
| `/channels` | Показать активные каналы, находящиеся на опросе |
| `/add <ссылка или ID>` | Добавить канал в список опроса |
| `/remove <ссылка или ID>` | Деактивировать канал (остановить опрос) |

### Команды полного доступа

| Команда | Описание |
|---------|-------------|
| `/allChannels` | Список всех каналов/групп, в которых состоит юзербот |
| `/privateChats` | Список личных чатов (исключая ботов) |
| `/addAdmin <роль> <id>` | Добавить администратора (`admin` или `full_access`) |
| `/removeAdmin <id>` | Удалить администратора |

## Планы по улучшению

Подробности о планируемом переходе на перцептивное хеширование см. в `docs/perception-hash-plan.md`, который позволит:

- Заменить MD5 на pHash/dHash для обнаружения визуального сходства
- Обнаруживать измененные по размеру, пережатые или слегка отредактированные версии одного изображения
- Использовать расстояние Хэмминга для настраиваемых порогов сходства
- Загружать миниатюры большего размера (тип `m`, ~320x320) для более надежного хеширования
